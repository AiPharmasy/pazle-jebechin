۱. معرفی بازی

«پازل جعبه‌چین» یک بازی فکری سبک سُوکوبان است که در آن بازیکن باید جعبه‌ها را به سمت نقاط هدف هل دهد. تمام ۱۵ مرحله بازی از مجموعه معروف Microban اثر David W. Skinner انتخاب شده‌اند — این مجموعه یکی از شناخته‌شده‌ترین مجموعه‌های سُوکوبان در جهان است که برای مبتدیان طراحی شده اما چالش‌های واقعی ارائه می‌دهد.

تمام مراحل با الگوریتم جستجوی BFS تأیید حل‌شدنی هستند و راه‌حل بهینه (کوتاه‌ترین) هر مرحله استخراج شده است. این راهنما شامل راه‌حل هر مرحله به فرمت LURD استاندارد است.

۲. قوانین بازی

  • بازیکن (دایره آبی با چهره) می‌تواند در ۴ جهت (بالا، پایین، چپ، راست) حرکت کند.
  • جعبه‌ها (مربع‌های نارنجی) فقط هل داده می‌شوند — بازیکن باید پشت آن‌ها بایستد و به سمت هدف هلشان دهد.
  • بازیکن نمی‌تواند جعبه را بکشد — فقط هل دادن مجاز است.
  • جعبه‌ای که به یک گوشه (دو دیوار مجاور) هل داده شود، دیگر قابل حرکت نیست — باید از دکمه «برگشت» استفاده کنید.
  • وقتی همه جعبه‌ها روی نقاط هدف (حلقه‌های طلایی) قرار گیرند، مرحله تمام می‌شود.
  • جعبه‌ای که روی هدف قرار گیرد سبز با تیک ✓ می‌شود.

۳. راهنمای نشانه‌های راه‌حل (LURD)

راه‌حل‌ها به فرمت استاندارد LURD نوشته شده‌اند که در جامعه سُوکوبان رایج است:

نشانه‌ها:
  • u = حرکت به بالا (بدون هل دادن جعبه) — Uppercase U = هل دادن جعبه به بالا
  • d = حرکت به پایین — D = هل دادن جعبه به پایین
  • l = حرکت به چپ — L = هل دادن جعبه به چپ
  • r = حرکت به راست — R = هل دادن جعبه به راست
نکته: تعداد کل حرکت‌ها = طول رشته LURD. تعداد هل دادن‌ها = تعداد حروف بزرگ. راه‌حل‌های ارائه‌شده بهینه هستند (کمترین تعداد حرکت ممکن).

۵. نصب APK روی گوشی

فایل‌های تحویلی

روش ۱: انتقال مستقیم (سریع‌ترین)

  1. فایل pazle-jebechin-v1.0-debug.apk را به گوشی خود منتقل کنید (بلوتوث، ایمیل، تلگرام، USB).
  2. در گوشی به Settings → Security بروید و گزینه Unknown Sources (منابع ناشناخته) را فعال کنید.
  3. فایل APK را در 파일 منیجر گوشی پیدا کرده و روی آن ضربه بزنید.
  4. روی Install ضربه بزنید و منتظر بمانید تا نصب کامل شود.
  5. آیکون «پازل جعبه‌چین» در منوی برنامه‌ها ظاهر می‌شود.

روش ۲: استفاده از ADB (برای توسعه‌دهندگان)

  1. ADB را نصب کنید (بخشی از Android SDK Platform-Tools).
  2. گوشی را به حالت Developer Mode ببرید و USB Debugging را فعال کنید.
  3. گوشی را با کابل USB به کامپیوتر وصل کنید.
  4. در ترمینال اجرا کنید:
adb install pazle-jebechin-v1.0-debug.apk

۶. انتشار در مارکت‌های ایرانی

گام ۱: ساخت keystore واقعی (یک بار برای همیشه)

هشدار مهم: keystore که برای انتشار استفاده می‌کنید باید برای همیشه نگه داشته شود. اگر آن را گم کنید، دیگر نمی‌توانید نسخه جدیدی از برنامه‌تان را در مارکت منتشر کنید. آن را در چند جای امن کپی کنید (cloud + فلش + هارد اکسترنال).

یک keystore واقعی با اطلاعات شخصی خودتان بسازید:

keytool -genkey -v \
  -keystore my-release-key.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias my-key-alias \
  -storepass YOUR_STORE_PASSWORD \
  -keypass YOUR_KEY_PASSWORD \
  -dname "CN=Your Name, OU=Game, O=Your Company, L=Tehran, ST=Tehran, C=IR"

گام ۲: امضای APK با keystore واقعی

ابتدا APK release بدون امضا را بسازید:

cd /home/z/my-project
bun run build:android
cd android
./gradlew assembleRelease

سپس با keystore خودتان امضا کنید:

# ابتدا zipalign
zipalign -v -p 4 \
  app/build/outputs/apk/release/app-release-unsigned.apk \
  app-release-aligned.apk

# سپس امضا
apksigner sign \
  --ks my-release-key.jks \
  --ks-key-alias my-key-alias \
  --ks-pass pass:YOUR_STORE_PASSWORD \
  --key-pass pass:YOUR_KEY_PASSWORD \
  --out pazle-jebechin-v1.0.apk \
  app-release-aligned.apk

# تأیید امضا
apksigner verify --print-certs pazle-jebechin-v1.0.apk

۷. ساخت مجدد APK پس از تغییرات

اگر کد بازی را تغییر دادید و می‌خواهید APK جدید بسازید:

گردش کار کامل

# ۱. تغییر کد در src/

# ۲. تست با dev server
bun run dev
# در مرورگر http://localhost:3000 را باز کنید

# ۳. لینت
bun run lint

# ۴. بیلد static export
bun run build

# ۵. همگام‌سازی با پروژه اندروید
npx cap sync android

# ۶. ساخت APK
cd android
./gradlew assembleRelease

# ۷. امضا (با keystore خودتان)
cd ..
bash scripts/sign-apk.sh

# ۸. APK نهایی در:
# download/pazle-jebechin-v1.0-release.apk

تنظیمات پیش از انتشار نسخه جدید

قبل از ساخت نسخه جدید، شماره نسخه را در فایل android/app/build.gradle افزایش دهید:

defaultConfig {
    applicationId "com.sokoban.puzzle"
    minSdkVersion rootProject.ext.minSdkVersion
    targetSdkVersion rootProject.ext.targetSdkVersion
    versionCode 2          // ← افزایش دهید
    versionName "1.1"      // ← افزایش دهید
}
قانون مارکت: هر بار که نسخه جدید در مارکت بارگذاری می‌کنید، versionCode باید از نسخه قبلی بزرگ‌تر باشد. versionName نام نمایشی است (مثل 1.0, 1.1, 2.0).

اعتبارسنجی مراحل پس از تغییر

اگر مراحل را تغییر دادید، قبل از ساخت APK حتماً این اسکریپت را اجرا کنید:

npx tsx scripts/extract-solutions.ts

این اسکریپت:

  1. تمام مراحل را با BFS حل می‌کند
  2. تعداد حرکات بهینه را محاسبه می‌کند
  3. راه‌حل هر مرحله را به فرمت LURD استخراج می‌کند
  4. فایل download/solutions.json را به‌روزرسانی می‌کند
  5. اگر مرحله‌ای قابل حل نباشد، هشدار می‌دهد
تست در مرورگر: برای تست راه‌حل‌ها در مرورگر قبل از ساخت APK، اسکریپت scripts/test-level.sh را اجرا کنید (نیازمند اجرای dev server روی پورت 3000).
